What do you do?

and you never go back

this is exactly how you do not use vim

normal mode is where you stay most of the time

every key is available

each key can represent a "word"

"words" can be combined into sentences

How does this work?

you just give it commands

like this

[cdyv=] [iatf] [w{"pt)sB]


easy

not really, but

vim is a language

actions are verbs


movements are nouns

verbs

an action we take


d: delete

c: change

y: yank

v: visually select

modifiers

to be used before a noun to describe the way in which you're going to do something


i: inside

a: around

NUM: number

t: search char and put the cursor before

f: search char and put the cursor ON it

/: find something (literal or regex)

nouns

objects you do something to


w: word

p: paragraph

}: paragraph (different way to do it)

t: tag (think html)

b: block

building sentences in this language?


d2w: delete two words

ct<: change until <

cit: change inside tag

yi": yank inside "

moving around?

hjkl (or arrows) are (almost) useless

there's usually a better way

moving in a line

0: beginning of line

$: end of line

^: first non blank character in line

t": jump to position just before next "

f": jump to position of next "

moving by words

w: next word word

b: back one word

e: end of word

uppercase variations ignore normal word delimiters (but space)

some other movements

ctrl-u: go up half a page

ctrl-d: go down half a page

ctrl-i: jump to your previous navigation location

ctrl-o: jump to where you were

gg: beginning of file

G: end of file

"rule of thumb"

if you're mashing something several times, there's probably a better way

the . command

repeats the last command

ex commands

:s/PATTERN/SUBSTITUTION/g

:g/PATTERN/command

:v/PATTERN/command

macros

when you try to quit vim and you miss ":" and it says "start recording" on the bottom

with available commands, vim's macros are awesome

start recording with qX

stop it with just q

execute it with @X
